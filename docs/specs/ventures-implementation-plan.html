Ventures Feature Implementation Plan

A handoff-ready spec for adding a private founder-equity and company history area to Finance Tracker. This feature is for one user only: it should be personal, visually impressive, and financially honest about private-company paper value.

Target route: /ventures Single-user personal feature Private equity is illiquid by default Use generated API contracts XY Flow React selected S3 + presigned URLs

Handoff Rules

Important: the reference images define visual direction, not exact requirements. Do not blindly copy fields, labels, currencies, or controls that do not make product sense. Every visible action must either work, be intentionally disabled with a reason, or be omitted until implemented.

Follow The Images For

  • Overall premium white/slate dashboard direction.
  • Large graph-first main screen with company cards.
  • Bottom sheet interaction pattern for quick inspect/create/edit.
  • Company workspace header, tabs, dense metric cards, and side panels.
  • Teal ongoing states, amber illiquidity, gold valuation/exits, muted red failure states.

Improve Or Correct

  • Inputs that are awkward, duplicated, or too narrow for real data.
  • Currency inconsistencies. Use SEK as the default unless the product later supports multi-currency.
  • Dead controls. Avoid decorative buttons that do nothing.
  • Generated labels that imply public-market liquidity.
  • The standalone “Lessons” entity. Use normal notes with tags instead.

Milestone 0 Decisions

Confirmed

  • Use @xyflow/react for the interactive graph and dagre for initial hierarchical layout.
  • Use react-markdown and remark-gfm for Markdown note rendering. The editor can start as a textarea with a small formatting toolbar.
  • These dependencies are temporarily ignored in apps/web/knip.json because M0 adds them before the feature imports them. Remove the ignores once implementation milestones consume the packages.
  • Use a dedicated AWS S3 bucket for company logos and documents. All upload/download access should go through presigned URLs.
  • Venture paper values stay inside the Ventures section only and must not affect the main net-worth, reports, or investment totals.
  • Risk-adjusted value is computed from paper value and haircut percentage.
  • Mobile is not a first-build requirement. Optimize for desktop.

Domain Decisions

  • Real company proceeds and values are not linked to ledger transactions in the first build.
  • Consulting/operating company valuations may be synced from linked Finance Tracker accounts, but the result is still shown only in Ventures.
  • Holding companies are supported: a company can own another company, so the graph is not limited to Casper-to-company edges.
  • Statuses are idea, ongoing, stale, exited, and failed. Do not add archived.
  • Notes are Markdown-capable and can use tags for lesson or reflection content. Do not create a separate Lessons model.
  • Document preview is out of scope for first build; upload and download are enough.

Product Scope

In Scope

  • Visual venture graph of Casper, holding companies, and owned/associated companies.
  • Company detail workspace with overview, timeline, valuation, notes, and documents.
  • Manual company creation with logo upload or initials fallback.
  • Manual valuation events including paper value, haircut percentage, and computed risk-adjusted value.
  • Account-linked valuation mode for operating/consulting companies where selected account balances drive the current venture value.
  • Ownership change tracking with effective dates and evidence.
  • S3-backed document/logo uploads and downloads via presigned URLs, plus document metadata tied to companies and events.
  • Markdown notes with tags such as lesson, risk, fundraising, board, and strategy.

Out Of Scope For First Build

  • Multi-user permissions, sharing, or collaboration.
  • Live cap-table import from third-party tools.
  • Automated private-market valuations.
  • Public market trading/price sync.
  • Separate “Lessons” data model. Use tagged notes instead.
  • Document preview, OCR, or full-text search. Upload/download and metadata are enough for the first build.
  • Mobile-specific layouts and mobile QA.
Keep normal investments and venture paper value separate. Private company paper value may be shown prominently in Ventures only, but it must not inflate liquid net worth, reports, or existing investment views.

UX Architecture

Main Graph Selected Company Panel Company Workspace Mutation Flows Timeline, Notes, Documents Desktop-Only First Build

Main Graph: /ventures

  • Top metrics: total paper value, risk-adjusted value, realized value, status counts, and illiquidity share.
  • Graph root is Casper, but companies can also own other companies for holding-company structures. Company cards branch by ownership relationship and saved layout.
  • Use XY Flow React for draggable nodes, edges, zoom controls, fit view, and custom node cards.
  • Connector lines should be cleaner than the earliest sketch: prefer straight, elbow, or restrained curved edges over loose wavy paths.
  • Nodes show logo/initial, company name, status, role, paper value, ownership, and latest update.
  • Graph positions should be persisted once the user moves nodes.

Selected Company Bottom Panel

  • Clicking a node opens a bottom sheet, not a side drawer.
  • Panel shows the selected company summary, valuation mini-chart, recent notes, documents, and quick actions.
  • Actions should include working routes/flows: Open company, Add valuation, Add note, Edit ownership.
  • Closing the panel should preserve graph state and selected node styling should clear.

Company Workspace: /ventures/:companyId

  • Header contains logo, name, status, role, ownership, paper value, risk-adjusted value, liquidity, confidence, and last updated.
  • Tabs: Overview, Timeline, Valuation, Notes, Documents. Do not add a separate Lessons tab in the first implementation.
  • Overview should summarize value history, ownership/risk profile, key details, recent notes, timeline highlights, and document health.
  • Use Recharts for valuation and ownership charts, matching the existing app chart approach.

Mutation Flows

  • Add company: bottom sheet from graph with logo upload, type/status/role, ownership, initial valuation, and live node preview.
  • Add annual valuation: bottom sheet from workspace with before/after preview, source, liquidity, confidence, and note.
  • Edit ownership: bottom sheet with current/new ownership, shares, share class, voting rights, evidence attachment, and timeline-event toggle.
  • Add note: small sheet/dialog with title, body, tags, optional linked event/document, Markdown body, and pinned flag.

Timeline, Notes, Documents

  • Timeline is an event feed grouped by year with filters for valuations, ownership, notes, documents, and milestones.
  • Notes replace the proposed Lessons concept. Notes can be tagged lesson or reflection when needed.
  • Documents should act as evidence for valuations and ownership, not just file storage.
  • Document health can be computed from required checklist items per company type and status.

Desktop-Only First Build

  • Desktop is the only required experience for the first implementation.
  • Do not spend milestone time on mobile-specific graph behavior, mobile sheets, or mobile table layouts.
  • Still avoid fixed-width choices that make later responsive work impossible.

Proposed Data Model

Names are suggestions. Keep the domain separate from existing investment accounts and snapshots.

venture_companies

  • id, user_id, timestamps.
  • name, legal_name, description.
  • company_type: startup, private_company, consulting, holding, other.
  • status: idea, ongoing, stale, exited, failed.
  • role: founder, co-founder, owner, advisor, investor, board observer, other.
  • valuation_mode: manual, account_balance_sync.
  • industry, stage, country.
  • founded_on, joined_on, exited_on, closed_on.
  • logo_storage_key, logo_file_name, logo_content_type. Serve logos through presigned read URLs or short-lived API URLs.
  • node_color, display_order.

venture_valuation_events

  • company_id, event_date, label, event_type.
  • paper_value_sek, haircut_percentage. Services compute risk-adjusted value as paper * (1 - haircut).
  • valuation_source: founder_estimate, financing_round, offer, annual_accounts, model, external_valuation, account_balance_sync.
  • liquidity_level: none, restricted, possible_secondary, liquid.
  • confidence_score: 1-5.
  • include_in_venture_totals; default true for current paper views.
  • note, linked document IDs.

venture_ownership_events

  • company_id, owner_type (person or company), optional owner_company_id, effective_date, reason.
  • direct_ownership_pct, fully_diluted_ownership_pct.
  • shares_owned, total_shares, share_class.
  • voting_rights_pct, option_or_warrant_notes.
  • invested_capital_sek where known.
  • note, linked document IDs.
  • Latest event derives current ownership shown in graph/workspace.

venture_timeline_events

  • company_id, event_date, event_type.
  • title, description.
  • Optional links to valuation, ownership, note, or document rows.
  • Generated automatically for core mutations, plus manual events.
  • Used by Timeline tab and selected-company panel.

venture_notes

  • company_id, title, body_markdown.
  • tags JSON/list: lesson, risk, board, fundraising, product, sales, strategy, finance.
  • pinned, note_date, timestamps.
  • Optional linked timeline event and document IDs.
  • No separate lessons table for first build.

venture_documents

  • company_id, title, document_type, category.
  • status: verified, linked, draft, pending_review, missing.
  • storage_key, external_url, file_name, mime_type, file_size_bytes.
  • Upload and download use presigned URLs. Document preview is not required for first build.
  • document_date, uploaded_at.
  • Optional linked valuation/ownership/timeline event.

venture_company_account_links

Link operating/consulting companies to existing Finance Tracker accounts for account-balance-driven valuations.

  • company_id, account_id.
  • include_in_synced_value, weight or percentage if partial balances are needed later.
  • Services derive the current paper value from linked account balances when valuation_mode is account_balance_sync.

venture_document_requirements

Optional but recommended for document health. Seed defaults in code rather than requiring user setup.

  • Company registration / articles of association.
  • Shareholder agreement, share register, and cap table.
  • Investment agreements, term sheets, and valuation memos.
  • Annual reports, management accounts, and financial models.
  • Board/shareholder minutes for major decisions.
  • Exit/closure evidence: SPA or sale agreement, closing memo, closure decision, final accounts, write-off memo.

venture_graph_layouts

Store graph layout independently from company data: company_id, x, y, pinned, viewport, and updated_at. This lets the user move nodes without corrupting business data.

API Contract Sketch

Method Path Purpose
GET /ventures/overview Graph KPIs, companies, latest valuations, latest ownership, saved layout, recent activity.
POST /ventures/companies Create company with optional initial ownership and valuation.
GET /ventures/companies/{companyId} Company workspace payload: summary, charts, timeline preview, notes, document health.
PATCH /ventures/companies/{companyId} Edit company metadata, status, role, dates, and logo reference.
POST /ventures/companies/{companyId}/valuations Add valuation event and generated timeline event.
POST /ventures/companies/{companyId}/ownership-events Add ownership change and generated timeline event.
GET/POST /ventures/companies/{companyId}/notes List/create notes. Tags cover lesson/reflection use cases.
GET/POST /ventures/companies/{companyId}/documents List/create document metadata for S3-backed documents.
PATCH /ventures/layout Persist XY Flow node positions and viewport.
POST /ventures/uploads/presign Create presigned S3 upload/download URLs for company logos and documents.
After changing routes or schemas, update infra/serverless/serverless.yml, apps/api/contracts/http.py, then run npm run generate:api-contracts and update frontend usage to generated types.

Resolved Decisions And Remaining Questions

Resolved In M0

  • Venture paper values are only shown inside Ventures and do not affect normal net worth, reports, or investments.
  • Risk-adjusted value is computed from paper value and haircut percentage.
  • Real company proceeds are tracked inside Ventures, not linked to ledger transactions in the first build.
  • Logos and documents use a new S3 bucket with presigned upload/download URLs.
  • Holding-company ownership is supported through company-to- company graph edges.
  • Statuses are idea, ongoing, stale, exited, and failed.
  • Notes support Markdown. Tagged notes replace a separate Lessons concept.
  • Logo upload/edit/remove is required. Document preview and mobile work are not required for the first build.

Still Open

  1. Exact S3 bucket name and lifecycle policy. Recommendation: private encrypted bucket with blocked public access and no automatic expiration for documents.
  2. Maximum file size and allowed MIME types. Recommendation: allow PDF, PNG/JPEG/WebP logos, XLSX/CSV, DOCX, and common image formats; reject executable/archive files.
  3. Whether document deletes should hard-delete from S3 immediately or soft-delete metadata first. Recommendation: soft-delete metadata, then optional cleanup job later.
  4. Whether account-linked valuation should include all account balances or only selected balance types. Recommendation: sum selected linked account current balances for M1.
  5. Whether Markdown notes need a full rich editor later. Recommendation: start with textarea + formatting toolbar + rendered preview using react-markdown.
  6. Whether valuation and ownership documents are mandatory blockers or health warnings only. Recommendation: warnings only.

Implementation Notes

  • Keep this feature separate from existing Investments; the data semantics are different.
  • Prefer real derived data over fake display values. If a metric cannot be computed, show an explicit empty/missing state.
  • Use succinct comments only around non-obvious valuation/ownership derivation logic.
  • Protect unrelated worktree changes. Stage only files that belong to the milestone being shipped.
  • When in doubt, choose financial honesty over visual excitement.
